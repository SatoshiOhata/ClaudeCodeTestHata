# UI/UXレビュースキル

HTMLファイルのUI/UX品質を評価し、改善提案を行う。

## チェック項目と基準

### 1. レスポンシブデザイン
- viewport metaタグが正しく設定されている
- メディアクエリでブレークポイントが定義されている
- モバイル・タブレット・デスクトップで崩れない
- 評価：3つ以上のブレークポイント → ✅ / 1〜2つ → ⚠️ / なし → ❌

### 2. カラーコントラスト
- テキストと背景のコントラスト比がWCAG AA基準（4.5:1以上）
- 重要なボタンやリンクが視認しやすい
- 評価：基準を満たす → ✅ / 一部不足 → ⚠️ / 大幅に不足 → ❌

### 3. タッチターゲット
- ボタンやリンクのサイズが44px x 44px以上
- タップ要素間に十分な間隔がある
- 評価：全て適切 → ✅ / 一部小さい → ⚠️ / 多数が小さい → ❌

### 4. フォントと可読性
- フォントサイズが16px以上（本文）
- 行間が1.5以上
- フォントファミリーが適切（システムフォント優先）
- 評価：全て適切 → ✅ / 一部不備 → ⚠️ / 読みにくい → ❌

### 5. アクセシビリティ
- lang属性が設定されている
- ARIA属性が適切に使われている
- キーボード操作が可能
- フォーカス表示がある
- 評価：主要項目対応 → ✅ / 一部欠落 → ⚠️ / 大幅に不足 → ❌

### 6. ビジュアル一貫性
- カラーパレットが統一されている
- 余白やサイズのルールが一貫している
- CSS変数で管理されている
- 評価：統一されている → ✅ / 一部不統一 → ⚠️ / バラバラ → ❌
