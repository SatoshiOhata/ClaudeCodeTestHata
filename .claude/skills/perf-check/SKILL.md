# パフォーマンスチェックスキル

HTMLファイルのパフォーマンスを分析し、最適化提案を行う。

## チェック項目と基準

### 1. ファイルサイズ
- HTML全体のサイズを測定
- インラインCSS/JSのサイズを個別に測定
- 目安：HTML 50KB以下 → ✅ / 50〜100KB → ⚠️ / 100KB超 → ❌

### 2. CSS最適化
- 未使用CSSの有無
- CSS変数の活用
- メディアクエリの効率性
- 評価：最適化済み → ✅ / 改善余地あり → ⚠️ / 問題あり → ❌

### 3. JavaScript最適化
- defer/async属性の使用
- 不要なDOMアクセスの有無
- イベントリスナーの効率性
- 評価：最適化済み → ✅ / 改善余地あり → ⚠️ / 問題あり → ❌

### 4. 画像最適化
- 画像フォーマット（WebP推奨）
- lazy loading の使用
- 適切なサイズ指定
- 評価：最適化済み → ✅ / 改善余地あり → ⚠️ / 未対応 → ❌ / 画像なし → N/A

### 5. レンダリングブロック
- CSSの読み込み方法
- JSの読み込み位置
- クリティカルCSSの分離
- 評価：ブロックなし → ✅ / 軽微 → ⚠️ / 深刻 → ❌

### 6. キャッシュ・プリロード
- preconnect/prefetchの使用
- リソースヒントの設定
- 評価：設定済み → ✅ / 一部設定 → ⚠️ / 未設定 → ❌
